Tutorial: Hosting A Small Business Website (Static) ON IBM Cloud

I deployed the site using a bucket in IBM’s Cloud Object Storage, and added a guest message board using Functions.
The process is super easy and you get a massive bang for your buck (In that you can deploy this for free).


I decided to create a dummy website for a BnB & hotel, you can view the hosted website here: http://tm-guestbook.s3-web.jp-tok.cloud-object-storage.appdomain.cloud/


I’ll give you a quick tutorial on how to get it done, and offer a review of my overall experience.

Create an IBM Cloudant Service

So What on earth is a Cloudant?  Here’s a super-quick summary.

IBM Cloudant is a distributed database that is optimized for handling heavy workloads that are typical of large, fast-growing web and mobile apps. You can download it straight from your IBM Cloud Catalog, or as an on-premises installation.

The API and replication protocol are compatible with an open source ecosystem that includes Apache CouchDB, PouchDB and libraries for the most popular web and mobile development stacks.

Ok….So what am I using it for?

In this tutorial, it’s used to create the database for the message board.

Create Functions

Next, you will have to create serverless actions (or Functions).

What is IBM Cloud Functions?

Based on Apache OpenWhisk, IBM Cloud™ Functions is a polyglot functions-as-a-service (FaaS) programming platform for developing lightweight code that scalably executes on demand.

How will I be using Functions?

You will be creating a sequence which is a chain of actions. In a sequence, the output of one action acts as input to the following action and so on.

5

Creating the First Action

1) In your IBM Cloud Console, Click on the menu dropdown and navigate to Functions

2) From the namespace drop-down, either select an existing namespace or use Create Namespace to create a new one.
3) On the side menu, select Actions
4) on the Actions page, click Create.

Previous
Next

5) Create a new action with the following settings

  • Name as prepare-entry-for-save
  • Click Create Package and set the package name as guestbook
  • Set Runtime as the latest Node.js
  • Click Create

 

6) In the page that appears next, add the below code and then click Save

/**
* Prepare the guestbook entry to be persisted
*/
function main(params) {
if (!params.name || !params.comment) {
return Promise.reject({ error: ‘no name or comment’});
}

return {
doc: {
createdAt: new Date(),
name: params.name,
email: params.email,
comment: params.comment
}
};
}

6

Add the Action To a Sequence:

On the same page, on the left select Enclosing Sequences and then Add To Sequence.

2) Set the new sequence name to save-guestbook-entry-sequence

3) Choose guestlist as the enclosing package and click Create and Add

Add a Second Action To a Sequence:

4) On the sequence page, click on the entry save-guestbook-entry-sequence. It opens sequence details. Then click Add on the upper right.
5)Instead of Create New select Use Public.

6) Pick Cloudant from the options loaded

  • Click Add and Pres Save

To test the entire sequence, click on Invoke with parameters and enter the JSON below and then click Invoke

 

{

7

Sequence of Actions to Retrieve Entries

1) Under Functions, click on Actions and then Create.

2) Create an Action with set-read-input as the name. Select guestbook as package and the latest version of Node.js as runtime.

3) Click Create.

4) In the action details, replace the existing code with the code snippet below. This action passes the appropriate parameters to the next action in the sequence.

function main(params) {
return {
params: {
include_docs: true
}
};
}

5) Click Save

8

Add the Action To a Sequence:

1)On the same page, on the left select Enclosing Sequences and then Add To Sequence.

2) Set the new sequence name to read-guestbook-entries-sequence

3) Choose guestlist as the enclosing package and click Create and Add

4) On the sequence page, Click on read-guestbook-entries-sequence sequence and then click Add to create and add the second action to get documents from IBM Cloudant.

5)Instead of Create New select Use Public.

6) Pick Cloudant from the options loaded

7) Under Actions, select List Document

8) Under Bindings select binding-for-guestbook and click Add to create and add this public action to your sequence.

9) Click Add again to add a third action. This one will format the documents from IBM Cloudant.

10) Under Create New enter format-entries as the name

11) Select guestbook for the Enclosing Package and then click Create and Add.

12) Click on format-entries and replace the code with below:
const md5 = require(‘spark-md5’); function main(params) { return { entries: params.rows.map((row) => { return { name: row.doc.name, email: row.doc.email, comment: row.doc.comment, createdAt: row.doc.createdAt, icon: (row.doc.email ? `https://secure.gravatar.com/avatar/${md5.hash(row.doc.email.trim().toLowerCase())}?s=64` : null) }}) }; }
13) Click on Save.

14) Choose the sequence by clicking on Actions and then read-guestbook-entries-sequence.

15) Click on Save and then Invoke.

Creating an API

2) Select the read-guestbook-entries-sequence sequence. Next to the name, click on Web Action, check Enable as Web Action and Save.

3) Do the same for the save-guestbook-entry-sequence sequence.

5) Set the API name to guestbook and, accordingly, the base path to /guestbook.

6) Click on Create operation and create an operation to retrieve entries with the following details:
  • Set path to /entries
  • Set verb to GET
  • Select the read-guestbook-entries-sequence action

Deploy The Website

Time for the fun part. Next you will be creating a Object Storage bucket configured with static website hosting containing the website files and the and message board javascript application that uses the cloudant database.

Follow the steps below

10

Create Cloud Object Storage

1) Click on Catalog and Search for Cloud Object Storage

2) Chose a unique service name for the instance. For this tutorial, I chose tm-guestbook-cos

3) Select the default resource group

4) Click Create

11

Create a Bucket

1) Click Create a bucket and select Customize Your Bucket

2) Enter a bucket name that is unique is across all ibm accounts. I used tm-guestbook

3) Choose the rest of your settings such as Resiliency and Location

4) Scroll down to the Static website hosting and click Add rule

5) Keep the Routing rules (individual) selected and add the Index document index.html

6) Click Public access to On and then click Save

7) Scroll to the bottom and click Create bucket

 

12

Copy Website Files

1) Downoad the following code from https://github.com/IBM-Cloud/serverless-guestbook

2) Edit guestbook.js – replace the value of apiUrl with the route you made a note of earlier when creating the API.

3) Copy the code in index.html and add it to your own index.html

4) Open the bucket Objects view and drag and drop the guestbook.js the index.html and the rest of your website files to the COS bucket

5) Navigate to the Configuration tab for the bucket and scroll down to the Static website hosting endpoints section to copy the Public endpoint into a browser tab

That's It. You should now be able to access your Website through your browser!

NOTE If you would like to link your own domain, Follow this tutorial: https://cloud.ibm.com/docs/cloud-object-storage?topic=cloud-object-storage-routing-rules-cos

How The IBM Cloud Performs

A review of my overall experience with the IBM Cloud service

Speed

As expected, the IBM Cloud  COS + Functions setup is pretty quick. Tested across multiple locations it performs well and generally better than the WordPress+ Kub setup (but not by much). Since this is a static site it loads pretty fast and you rarely have any lag
