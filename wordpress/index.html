Setting Up Your Kubernetes Cluster

So What on earth is Kubernetes?  Here’s a super-quick summary.

Kubernetes, also known as K8s, is a portable, extensible, open-source platform for managing containerized workloads and services, that facilitates both declarative configuration and automation. It groups containers that make up an application into logical units for easy management and discovery.

Ok….But what are containers??

Containers are a method of building, packaging and deploying software. Typically, a container would include everything the containerized workload needs to run, including all the code, runtime and libraries.

Similar to a virtual machine, a container has its own filesystem, share of CPU, memory, process space, and more. (See the image below)

To read more about Kubernetes, click here.

Install Block Storage Plugin

IBM Cloud Block Storage is persistent, high-performance iSCSI storage that is provisioned and managed independently of compute instances. iSCSI-based Block Storage LUNs are connected to authorized devices through redundant multi-path I/O (MPIO) connections.

Install the IBM Cloud Block Storage plug-in with a Helm chart to set up pre-defined storage classes for Block Storage. You’ll use these storage classes to create a PVC to provision Block Storage for your app.

WAAAAIT….What is a PVC?

A PersistentVolume (PV) is a piece of storage in the cluster that has been provisioned by an administrator or dynamically provisioned using Storage Classes. It is a resource in the cluster just like a node is a cluster resource. PVs are volume plugins like Volumes, but have a lifecycle independent of any individual Pod that uses the PV. This API object captures the details of the implementation of the storage, be that NFS, iSCSI, or a cloud-provider-specific storage system.

A PersistentVolumeClaim (PVC) is a request for storage by a user. It is similar to a Pod. Pods consume node resources and PVCs consume PV resources. Pods can request specific levels of resources (CPU and Memory). Claims can request specific size and access modes (e.g., they can be mounted ReadWriteOnce, ReadOnlyMany or ReadWriteMany, see AccessModes).

Got it? if not, read more here

Go ahead and install the Block Storage plugin by following these steps.

4

Open Up IBM Cloud Shell

Click on the IBM Cloud Shell icon on the top right corner of your console. The IBM Cloud Shell is IBM’s web-based version of PoweShell. It makes running commands so much easier.

5

Target The Correct Cluster

Use the following command to target the correct cluster 

ibmcloud ks cluster config –cluster [Your Cluster ID]

(Replace [Your Cluster ID] with your actual cluster ID which you can find on your Cluster page here.

6

Plugin Installation

1) Use the following command to add the IBM Cloud Helm chart repository to the cluster where you want to use the IBM Cloud Block Storage plug-in.

helm3 repo add iks-charts https://icr.io/helm/iks-charts

2) Update the Helm repo to retrieve the latest version of all Helm charts in this repo.

helm3 repo update

3) Install the IBM Cloud Block Storage plug-in and give your installation a name

helm install [name] iks-charts/ibmcloud-block-storage-plugin -n [namespace]

  • Replace [name] with your desired name
  • Replace [namespace] with the word ‘default’

4) Verify that the installation was successful. (Status should show as running)

kubectl get pod -n default

 

NOTE

Do not close cloud shell yet, you will need it shortly.

Install WordPress

WordPress is a free and open-source content management system written in PHP and paired with a MySQL or MariaDB database. It is the worlds most popular CMS powering more than 30% of the websites on the internet.

You should now be ready to deploy WordPress onto the cluster. The installation will be done straight from the catalog however, there are a few steps that will be required to actually access your website.

7

Navigate To WordPress In the Catalog

Click on Catalog and Search for WordPress 

8

Setup Your Installation

On this page, you will be able to set up the various aspects of your WordPress Installation.

Set the installation up as follows:

  • Target – IBM Kubernetes Service
  • Method – Helm Chart
  • Cluster – Your desired cluster
  • Target Namespace – Default

You can also name your workspace and set the deployment values

9

Open The Workspace Log

Once WordPress has been installed, open up the log by clicking “View Log“. You can also navigate here by going to menu -> Schematics -> Workspaces -> Your WordPress installation.

NOTE
Viewing the log online may be frustrating sometimes as it refreshes every few seconds. To avoid pulling out your hair, Copy the log and view it in WordPad (With word wrap set to wrap to window)

10

Run Commands to access your WordPress site.

In the log, look for the following text” To access your WordPress site from outside the cluster follow the steps below:“

The lines that follow in the log are a set of commands you will need to run in Cloud Shell in order to get your site URL, username and password (If you didn’t set them when installing). 

Run the commands as follows: 

1) run the command that looks similar to this (Yours will be slightly different). The command returns no output

export SERVICE_IP=$(kubectl get svc –namespace default wordpressa0-145c-4f –template “{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}”)

2) To retreive your website url, run the following command

echo “WordPress URL: http://$SERVICE_IP/”  

3) To retrieve your website login page URL, run the following command

echo “WordPress Admin URL: http://$SERVICE_IP/admin””  

That's It. You should now be able to access your WordPress site through your browser!

11

Accessing Your Database

Your WordPress installation will be paired with a MariaDB database. To connect to your database, retrieve your database username and password (If you didn’t set them up when deploying WordPress, use the commands in the log file to get them) and then do the following:

1) Run a pod that you can use as a client. Run the command that looks similar to this (Yours will be slightly different). 

kubectl run wordpressa0-145c-4f-mariadb-client –rm –tty -i –restart=’Never’ –image docker.io/bitnami/mariadb:10.5.8-debian-10-r0 –namespace default –command — bash  

2) To connect to primary service (read/write), run the command that looks similar to this (Yours will be slightly different).

mysql -h wordpressa0-145c-4f-mariadb.default.svc.cluster.local -uroot -p bitnami_wordpress

You will be prompted for your database username. ANd then can access your database.

How The IBM Cloud Performs

A review of my overall experience with the IBM Cloud service

Speed

As expected, the IBM cloud + Kubernetes setup is pretty quick. Tested across multiple locations it performs well.

My website isn’t a resource-intensive one such as an e-commerce site, but I expect that even if yours is, you will get better performance than you would on the common shared hosting servers offered by companies like GoDaddy or BlueHost.

My Rating:
 4/5

Reliability

Well, this is a bit of a no-brainer. IBM servers are infamous for their reliability. From AS400 to zSeries, you’re pretty much guaranteed a super reliable (albeit bank-breaking) machine that won’t let you down.

So while I have only tested the cloud service for a week, I expect it to be as reliable as all their physical machines.

My Rating: Provisional
 5/5

Usability

Ok, so here it gets a bit complicated. The usability is almost entirely dependant on whether or not you are familiar with cloud servers & command-line environments.

If you don’t have knowledge of the above and all you’re used to is cPanel, Plesk or some GUI type of environment….well, you’re going to be confused as hell with the IBM Cloud Server, especially when something goes wrong or doesn’t work.

On the other hand, if you’re comfortable with the above, then the setup will be easier for you.
My Rating:
 2.5/5

Price

While there is a massive value for money (given how much you can get done with the free IBM Cloud) It’s an IBM service soooo…It’s a bit expensive.

Okay you won’t have to sell a kidney, and this will only happen if your usage is high. However, if you want to get some real work done and go beyond what the free plan can offer, you’re going to have to part with some bucks. But it’s generally worth it if you use it to it’s full potential.
My Rating:
 3.5/5

Customer Support

The customer support offered by the IBM team is exceptional. This was a huge standout for me, as their support team was polite, experienced and patient. They should really consider outsourcing their support staff to other companies.

My Rating:
 5/5

Conclusion

Overall the experience of hosting a website on the IBM Cloud has been a pleasant one. Their support team was exceptional and for me, this makes or breaks a service. If you’re looking to host your business on a fast, reliable and powerful system, I would recommend it. Just make sure your budget allows.
